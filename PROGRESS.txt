Phase 1 — Terrain & Physics
  WA-sourced terrain generation (generate_tactical — the sole live generator): macro
    silhouette now comes from two real Worms Armageddon terrain bitmaps extracted from
    the original game's land.dat, baked as Rust constants (src/world/wa_templates.rs,
    include_bytes!, no runtime I/O); seed picks mask/x-shift/mirror deterministically (v0.5.4.392)
  5-way archetype system removed (v0.5.4.393): replaced by template_id (which of the 2 WA
    masks) + is_cavern (occasional carved-cavern mode, ~20% odds). Chasms, overhang shelves,
    and cave carving are now seed-random on any non-cavern map instead of archetype-gated.
  World height doubled to 960px (2× screen); maps use full vertical range with peaks
    near the top and caves/canyons running deep; cave constants scaled proportionally (v0.5.4.387)
  Vertical spawn spread (v0.5.4.389): soldiers spawn at varied heights — maps with punched
    caves reserve half the spawn slots for underground positions so teams start on ledges,
    in tunnels, and at mid-terrain, not just on the topmost surface layer
  Destructible terrain (crater carving persists)
  Projectile ballistics (Euler integration, wind, gravity, bounce/friction)
  Water death zone

RECENT CHANGES (WA terrain art + archetype removal + bugfixes, v0.5.4.392–.395, 2026-07-01)
--------------------------------------------------------------------------------------------
  v0.5.4.392: Terrain silhouettes sourced from real Worms Armageddon terrain art.
    Two 1-bit-per-pixel, 1920x696 terrain masks extracted directly from land.dat
    (reverse-engineered format: LND magic header, index table, IMG chunks). Baked
    as Rust constants via include_bytes! — generate_tactical stays pure/seed-derived,
    no runtime file I/O. Seed selects mask (0/1), x-shift, and mirror.
  v0.5.4.393: Removed the 5-way archetype system entirely. terrain.archetype (u8)
    replaced by template_id (u8, which WA mask) + is_cavern (bool, ~20% odds,
    independent carved-rock generation path). Chasms/overhangs/caves decoupled
    from archetype — now seed-random on any non-cavern map, so combinations that
    were previously impossible (e.g. chasms + overhangs together) can now occur.
    Cosmetic dispatch (sky color, background debris, dirt tint, scenery theme)
    collapsed from 5-way to 3-way: is_cavern, template_id 0, template_id 1.
  v0.5.4.394: Fixed aim angle rotating while panning camera with R1. process_aim()
    only excluded L1 (which also pans the camera) from Up/Down aim adjustment;
    R1 pans vertically too and was missed. Both now excluded.
  v0.5.4.395: Fixed invisible soldiers. render_my_team()'s vertical draw-cull for
    soldiers/headstones/projectiles/explosions checked world-y against a hardcoded
    0..480 window instead of the camera-scrolled viewport (cam_y..cam_y+480) — any
    soldier spawned below y=480 in the 960px world was silently never drawn. Got
    much more common after the WA-terrain rewrite since spawns land deeper more
    often (especially cave floors). Fixed all four cull sites to use cam_y.

RECENT CHANGES (terrain shape, v0.5.4.385–.386, 2026-06-30)
-----------------------------------------------------------
  Terrain silhouettes made smooth/organic instead of jagged. In generate_tactical the
    continuous density field is now box-blurred before thresholding (archetypes hills/
    cliffs/islands/canyon); blurring the continuous field rounds contours without eroding
    thin bridges/stepping-stone islands. Caverns (archetype 3) unchanged.
  .385 = conservative (blur radius 3); .386 = much rounder (radius 10, islands 6; fine
    relief octave dropped, ×4 halved, threshold lowered to offset blur shrinkage).
  Tradeoff: gentler slopes → more terrain is walk-up-able (the fine relief that forced
    jump/backflip is largely gone) — accepted for the look.
  Deterministic (no new RNG, fixed-order f64) so client/server rebuild identical terrain
    from the seed; no StateMsg/parity change.
  Legacy generate_worms generator (old full-Perlin jagged style) deleted — generate_tactical
    is now the ONLY generator reachable in-game (from_heightmap remains test-only).

KEY ARCHITECTURE NOTES
-----------------------
  simulate_with_muzzle(game, input, muzzle, aim_angle) — single sim function
    called by all 5 paths. aim_angle: Option<f32> lets server pass received angle
    directly; Up/Down buttons are never stripped and flow to all weapons.
  StateMsg round-trip — live client rebuilds state from server's StateMsg each tick.
    Never runs simulate. Enforced by compile-time checklists in net_sync.rs.
  emit_fx / emit_sound — all particle FX and sounds go through these channels
    to auto-replicate to live clients. Never call fx functions directly from sim.
  assert_all_paths_in_sync — parity test helper in tests/parity.rs. Call for
    every new gameplay feature test.
  Framebuffer blit (Miyoo): rows reversed into heap flip_buf, flushed to mmap
    in one copy per frame. Screen is 640x480, rotated 180° in hardware.
